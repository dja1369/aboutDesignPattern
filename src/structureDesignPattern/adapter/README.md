# 어댑터 
1. > 어댑터 패턴은 서로 다른 인터페이스를 가진 두 객체를 연결하는 디자인 패턴이다.
2. > 어댑터 패턴은 기존의 클래스를 수정하지 않고, 새로운 인터페이스에 맞게 클래스를 수정하는 방법을 제공한다. 
3. > 이를 통해 기존의 클래스를 재사용할 수 있다.
4. > - 단일책임원칙: 기존의 비즈니스 코드에서 다른 호환되지 않는 비즈니스 코드를 연결하는 역활로 객체간의 역활을 분리 할 수 있다.
   > - 걔방폐쇄원칙: 클라이언트가 어댑터를 통해 작동하는한 기존의 비즈니스 코드를 수정할 필요가 없다.
   > - 예: 110V 전기를 사용하는 제품을 220V 컨버터에 연결해 사용하는 것과 같다.
